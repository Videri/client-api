Vistar Media Client API

Provided is the Vistar Media client API. It's assumed from here on out that you have an active account on Vistar Media's system which will provide you with a network identifier and an API key.

For more information on obtaining an API key, email api@vistarmedia.com

Overview

The provided API allowes a network to serve real time ads on their devices and submit the necessary proofs of play after the fact. It is meant to integrate into a Java CMS, Scheduler, or player.

Project Integration

Depending on your project's layout, there are a number of ways to integration Vistar Media's API. A few are outlined below.

Maven Integraiton

As there are quite the grab bag of HTTP clients available on the JVM, there are a number of implementations provided, each scoped {@code provided}. You must choose the HTTP client below you'd like to use and not only follow the specified configuration, but ensure that it's available in your build (the dependency will not be specified here). 

<dependencies>
  <dependency>
    <groupId>com.vistarmedia</groupId>
    <artifactId>api</artifactId>
    <version>1.1-SNAPSHOT</version>
  </dependency>
</dependencies>
Async HTTP Client

The Reccomended HTTP client to use is async-http-client. This will allow for bulk requests to be made in parallel. The downside is that it pulls in potentially conflicting dependencies (such at Netty).

To use async-http-client, add the following to your project's {@code pom.xml}: 

<dependencies>
  <dependency>
    <groupId>com.ning</groupId>
    <artifactId>async-http-client</artifactId>
    <version>1.6.5</version>
  </dependency>
</dependencies>
